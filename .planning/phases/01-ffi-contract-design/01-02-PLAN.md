<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Go / purego callers -> C ABI | Incorrect calling-convention choices would become runtime memory corruption across five targets |
| Rust shim contract -> later C++ implementation | Ambiguous ownership or stale-handle rules would become use-after-free and invalidation bugs later |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | public function signatures | mitigate | Keep every prototype on the lowest-common-denominator ABI shape: `int32_t` returns, pointer out-params, no struct-by-value, no scalar float/int mixing |
| T-02-02 | T | handle and value model | mitigate | Encode packed handle parts plus doc-tied view/iterator structs directly in `src/lib.rs` so stale-handle and view-lifetime rules are fixed in the contract surface |
| T-02-03 | D | parser lifecycle surface | mitigate | Comment and encode explicit parser-busy semantics in the exported contract so later phases cannot implement implicit doc invalidation |
</threat_model>
