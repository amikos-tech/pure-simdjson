## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| workflow env -> target-tag smoke process | Job inputs and env vars determine which published source tree and source ladder are exercised, and where bootstrap bytes are cached | `version`, `goos`, `goarch`, `mode`, cache-dir |
| public artifact bytes -> local cache | Public `SHA256SUMS` and artifacts from R2 / GitHub Releases land on disk before the bootstrap path loads them | checksum metadata, shared-library bytes |
| current-branch workflow files -> target-tag source checkout | The validation workflow runs from the current branch while checking out the published tag into `target-src/` | workflow YAML, wrapper script, tag ref, checked-out source tree |
| public R2 -> GitHub fallback ladder | Validation must prove an intentional GitHub fallback path rather than an accidental R2 success | mirror base URL, HTTP `404` probe results, fallback env vars |
| workflow/docs -> operator action | Operators rely on the runbook to dispatch the correct workflow against the intended published tag | workflow filename, rerun instructions, matrix scope |
