- after `release.yml` succeeds for `v<version>`, operators run `public-bootstrap-validation.yml` with `version=<semver-without-v>`
- the workflow itself stays on the current branch, but checks out the requested tag into `target-src` so reruns validate the published source tree instead of whatever the branch looks like later
- the workflow runs:
  - full-matrix R2-only validation on linux/amd64, linux/arm64, darwin/amd64, darwin/arm64, windows/amd64

- operators should inspect the workflow summary for mode, target, expected cache path, unix perms, and broken-mirror probe evidence
- this is the canonical Phase `06.1` entrypoint and remains separate from the publish workflow

- that live validation is owned by Phase `06.1` through `public-bootstrap-validation.yml`
- that live validation runs against `target-src` from the published tag rather than the moving current branch source
