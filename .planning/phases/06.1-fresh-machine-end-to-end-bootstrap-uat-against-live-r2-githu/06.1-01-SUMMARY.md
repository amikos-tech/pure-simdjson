# Phase 06.1 Plan 01 Summary

**Added a reusable live-public bootstrap wrapper that forces either the R2 path or the GitHub Releases fallback path, clears an isolated cache safely, proves the expected artifact lands in `v<version>/<goos>-<goarch>/<libname>`, and records unix permission evidence for CI summaries**

## Accomplishments

- Created `scripts/release/run_public_bootstrap_smoke.sh` with strict argument parsing for `--repo-root`, `--version`, `--goos`, `--goarch`, `--mode`, and `--cache-dir`.
- Added cache-deletion guardrails that refuse empty values, `.`, `..`, `/`, Windows drive roots, the resolved repo root, and the resolved `$HOME` path before `rm -rf`.
- Implemented portable unix permission checks via `stat -c '%a'` with `stat -f '%Lp'` fallback so linux and macOS runners both enforce the locked `0700` contract.
- Added deterministic `r2` and `github-fallback` mode handling, including broken-mirror `404` proof for both `SHA256SUMS` and the target artifact before fallback runs.
- Wrote stable `GITHUB_STEP_SUMMARY` output with mode, target, expected cache path, fallback evidence, and observed unix permissions.

## Decisions

- Kept the wrapper self-contained instead of importing Go helpers so workflow steps can validate any published tag from a separate checkout.
- Rejected dangerous cache directories in shell before any deletion because CI runners still execute `rm -rf` as the current user.
