  <action>
Create `scripts/release/run_public_bootstrap_smoke.sh` as a strict `bash` wrapper with `set -euo pipefail`, `usage()`, and small helpers for path normalization, safe cache deletion, portable permission reads, and GitHub step-summary output.

The script must require these exact arguments:
- `--repo-root <path>`
- `--version <semver-without-v>`
- `--goos <linux|darwin|windows>`
- `--goarch <amd64|arm64>`
- `--mode <r2|github-fallback>`
- `--cache-dir <path>`

Implement the exact library-name mapping locally in the script:
- `linux` -> `libpure_simdjson.so`
- `darwin` -> `libpure_simdjson.dylib`
- `windows` -> `pure_simdjson-msvc.dll`

Before running the smoke:
- resolve `repo_root` and `cache_dir` to absolute paths
- hard-fail if `--repo-root/tests/smoke/go_bootstrap_smoke.go` is missing
- add `refuse_unsafe_cache_dir()` that rejects at minimum:
  - empty values
  - `.`
  - `..`
  - `/`
  - Windows drive roots such as `C:/`
  - the resolved `repo_root`
  - the resolved `$HOME`
- only after that guard passes, remove the requested cache dir completely
- recreate nothing by hand beyond temporary bookkeeping; the bootstrap path must create the cache directories itself

Add a portable `stat_mode()` helper that:
- first tries `stat -c '%a'`
- then falls back to `stat -f '%Lp'`
- fails with a clear message if neither form works

For `--mode r2`:
- `unset PURE_SIMDJSON_BINARY_MIRROR`
- `export PURE_SIMDJSON_DISABLE_GH_FALLBACK=1`

For `--mode github-fallback`:
- `export PURE_SIMDJSON_BINARY_MIRROR=https://releases.amikos.tech/pure-simdjson-does-not-exist`
- `unset PURE_SIMDJSON_DISABLE_GH_FALLBACK`
- compute:
  - `broken_checksums_url="$PURE_SIMDJSON_BINARY_MIRROR/v$version/SHA256SUMS"`
  - `broken_artifact_url="$PURE_SIMDJSON_BINARY_MIRROR/v$version/$goos-$goarch/$libname"`
- probe both URLs with `curl -sS -o /dev/null -w '%{http_code}'`
- require both probes to return HTTP `404` before smoke runs; otherwise fail with an explicit message that the fallback proof is ambiguous

For both modes:
- `export PURE_SIMDJSON_CACHE_DIR="$cache_dir"`
- run `(cd "$repo_root" && go run ./tests/smoke/go_bootstrap_smoke.go)`

After the smoke succeeds:
- compute `expected_cache_path="$cache_dir/v$version/$goos-$goarch/$libname"`
- fail if that file does not exist
- on non-windows targets, assert `stat_mode()` returns exactly `700` for:
  - `"$cache_dir"`
  - `"$cache_dir/v$version"`
  - `"$cache_dir/v$version/$goos-$goarch"`

If `GITHUB_STEP_SUMMARY` is set, append a stable markdown section headed `## public bootstrap validation` containing:
- mode
- target
- expected cache path
- whether GitHub fallback was enabled
- broken-mirror checksum/artifact probe statuses when applicable
- observed unix perms for the checked directories when applicable
