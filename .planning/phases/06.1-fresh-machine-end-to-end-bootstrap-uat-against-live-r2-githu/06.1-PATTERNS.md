## Pattern Assignments

### `.github/workflows/public-bootstrap-validation.yml` — CREATE

**Analogs**

- `.github/workflows/release.yml` — pinned action SHAs, runner labels, matrix style
- `.github/workflows/phase3-go-wrapper-smoke.yml` — simple cross-platform Go smoke layout

**What to keep**

- runner labels already used by the repo:
  - `ubuntu-latest`
  - `ubuntu-24.04-arm`
  - `macos-15-intel`
  - `macos-15`
  - `windows-latest`
- pinned `actions/checkout` and `actions/setup-go` SHAs from `release.yml`
- `defaults.run.shell: bash`

**What to adapt**

- trigger is `workflow_dispatch`, not `push.tags`
- workflow validates already-published artifacts instead of building anything
- current branch checkout and target-tag checkout must be separate
- R2 full matrix and GitHub fallback subset must stay in different jobs

### `scripts/release/run_public_bootstrap_smoke.sh` — CREATE

**Analogs**

- `scripts/release/run_go_packaged_smoke.sh` — wrapper that prepares env and runs the Go smoke binary
- `scripts/release/run_alpine_smoke.sh` — one-purpose shell contract with strict argument handling

**What to keep**

- strict `set -euo pipefail`
- explicit `usage()` function
- hard-fail on missing required args
- write human-readable failure messages

**What to adapt**

- live public validation uses a real published tag checkout, not a locally staged mirror tree
- cache path and unix perms must be asserted after success
- two explicit modes:
  - `r2`
  - `github-fallback`

### `scripts/release/test_public_bootstrap_validation_contracts.py` — CREATE

**Analog**

- `scripts/release/test_release_workflow_contracts.py`

**Pattern**

- use `pathlib.Path(...).read_text()` + `unittest`
- assert workflow structure by locating concrete strings
- assert the shell script still contains the forced-mode env vars and cache-path formula

## Planner Guidance

- Keep the validation workflow completely separate from `release.yml`.
- Validate the requested published tag from a `target-src/` checkout instead of relying on the current branch's `Version` constant.
- Force the source ladder explicitly with env vars rather than adding temporary code branches.
- Treat `internal/bootstrap/checksum_resolution.go` as the production contract: `SHA256SUMS` is the live metadata source.
- Prefer grep-testable workflow and script contracts over brittle local integration tests that try to emulate GitHub-hosted runners.
