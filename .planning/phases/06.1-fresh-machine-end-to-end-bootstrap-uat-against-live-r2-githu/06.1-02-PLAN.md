<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| current-branch workflow files -> target-tag source checkout | validating the wrong source tree invalidates the whole phase |
| public R2 -> GitHub fallback ladder | the workflow must prove each source path intentionally, not accidentally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06.1-02-01 | T | target source selection | mitigate | workflow checks out `target-src` from `refs/tags/v${{ inputs.version }}` and validates `go.mod`, `internal/bootstrap/version.go`, and the smoke file before setup or smoke |
| T-06.1-02-02 | T | runner shell drift | mitigate | both jobs force `defaults.run.shell: bash` and contract tests lock that in |
| T-06.1-02-03 | T | unbounded runner hangs | mitigate | both jobs use `timeout-minutes: 20` so stuck network or checkout issues fail predictably |
| T-06.1-02-04 | T | R2 full matrix | mitigate | `r2` mode disables GitHub fallback completely so an R2 outage cannot be masked |
| T-06.1-02-05 | T | GitHub fallback subset | mitigate | fallback mode forces the mirror base to a guaranteed-404 path and the wrapper proves that before smoke runs |
</threat_model>
