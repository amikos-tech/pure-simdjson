## Fault Injection Test Matrix

| Fault | Test Pattern | Expected Behavior | Requirement |
|-------|--------------|-------------------|-------------|
| requested version does not match the checked-out target tag source | workflow checks `target-src/internal/bootstrap/version.go` | job fails before smoke | UAT-06.1-01 |
| stale cache makes the bootstrap path look green | wrapper removes `PURE_SIMDJSON_CACHE_DIR` before `go run` | job fails unless the artifact is downloaded again | UAT-06.1-04 |
| R2 is down but the full matrix still passes through fallback | `PURE_SIMDJSON_DISABLE_GH_FALLBACK=1` in `r2` mode | job fails instead of silently masking an R2 outage | UAT-06.1-02 |
| GitHub fallback subset accidentally still uses R2 | broken mirror base `https://releases.amikos.tech/pure-simdjson-does-not-exist` | job only passes if GitHub fallback works | UAT-06.1-03 |
| unix perms drift from `0700` while parse still succeeds | wrapper stats cache dirs after smoke | unix jobs fail on wrong perms | UAT-06.1-05 |

## Security Domain

### Applicable ASVS L1 Categories

| ASVS Category | Applies | Standard Control |
|---------------|---------|------------------|
| V1 Architecture | yes | separate post-publish workflow; do not fold into `release.yml` |
| V5 Input Validation | yes | version input, target-tag checkout, and explicit target-source verification |
| V6 Cryptography | yes | live validation depends on published `SHA256SUMS`, not local overrides |
| V14 Build / Deploy | yes | fresh-runner jobs validate the actual released bytes and cache installation path |

### Known Threat Patterns

| Pattern | STRIDE | Standard Mitigation | Test |
|---------|--------|---------------------|------|
| stale cache produces false-positive success | Tampering / Quality gap | always clear `PURE_SIMDJSON_CACHE_DIR` | UAT-06.1-04 |
| wrong tag/source tree validated after the repo advances | Tampering | dual checkout with `target-src` version guard | UAT-06.1-01 |
| GitHub fallback masks an R2 outage in the full matrix | Availability gap | disable fallback for `r2` mode | UAT-06.1-02 |
| local checksum overrides hide live metadata regressions | Tampering | use published `SHA256SUMS` path only | UAT-06.1-02 / UAT-06.1-03 |
