<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source version -> published artifact URL | `bootstrap.Version` determines which immutable release tree default installs fetch. |
| Go wrapper ABI -> Rust native ABI | Loader must reject incompatible artifacts and readiness must catch source drift before publication. |
| release notes source -> GitHub release notes | `CHANGELOG.md` is rendered into public release notes by `release.yml`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09.1-01 | Tampering | ABI constants | mitigate | The new checker requires Go and Rust ABI constants to match exactly. |
| T-09.1-02 | Spoofing | bootstrap release pin | mitigate | ABI policy rejects `bootstrap.Version` lower than the minimum known version for ABI `0x00010001`. |
| T-09.1-03 | Tampering | checksum source | mitigate | Keep `Checksums` empty and rely on published `SHA256SUMS`; acceptance checks reject accidental embedded source digests. |
| T-09.1-04 | Repudiation | release notes | mitigate | Promote dated `0.1.2` changelog section before tag publication so rendered notes match source state. |
| T-09.1-09 | Tampering | development-time ABI drift | mitigate | `internal/bootstrap/abi_assertion.go` uses compile-time array-length assertions so `go test ./internal/bootstrap` fails if `ffi.ABIVersion` no longer matches the ABI expected by bootstrap version `0.1.2`. |
</threat_model>
