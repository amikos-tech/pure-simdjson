## ABI-drift prevention policy

| Option | Description | Selected |
|--------|-------------|----------|
| check_readiness.sh assertion + build-time canary | Extend --strict with ABI-major vs bootstrap-version cross-check; add trivial Go build-time assertion as zero-maintenance canary | ✓ |
| check_readiness.sh assertion only | Pre-tag shell gate only | |
| Defer entirely to a hardening phase | Ship Phase 09.1 with alignment only; add prevention later | |
| CI-side ABI consistency job (dlopen) | Post-publish regression monitor; high complexity | |

**User's choice:** check_readiness.sh assertion + build-time canary (Recommended)
**Notes:** Root cause was a manual step: Phase 8 bumped ffi.ABIVersion but version.go wasn't updated. The check_readiness.sh assertion catches this at release time; the Go build-time canary catches it during development (failing go build if the pin is stale).
