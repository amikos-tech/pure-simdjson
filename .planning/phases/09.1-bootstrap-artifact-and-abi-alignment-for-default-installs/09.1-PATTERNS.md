# Phase 09.1 Pattern Map

## Source Release-State Gate

**New file:** `scripts/release/check_bootstrap_abi_state.py`

**Closest analogs:**
- `scripts/bench/check_benchmark_claims.py` - standard-library Python CLI, deterministic JSON/text parsing, fail-closed validation.
- `scripts/release/render_release_notes.py` - release-facing Python CLI style.
- `scripts/release/check_readiness.sh` - shell entrypoint that delegates focused checks and exits nonzero before tag publication.

**Pattern to follow:**
- Use `argparse`, `pathlib`, `re`, and simple semver tuple parsing.
- Emit direct stderr diagnostics and nonzero exits for release-blocking state.
- Keep the helper independent of network and git state; `check_readiness.sh` already owns `origin/main` ancestry.

## Readiness Shell Integration

**Modify:** `scripts/release/check_readiness.sh`

**Closest analog:** Existing strict-mode checks in the same file.

**Pattern to follow:**
- Only run expensive/release-specific checks when `--strict` is set.
- Pass `--version "$version"` to the Python helper after `bootstrap.Version` is resolved.
- Preserve the existing final success line shape.

## Bootstrap Version Source

**Modify:** `internal/bootstrap/version.go`

**Closest analog:** Current file; it is the single source of truth.

**Pattern to follow:**
- Keep a constant string, not ldflags.
- Exact target state for Phase 09.1 is `const Version = "0.1.2"`.

## Build-Time ABI Canary

**New file:** `internal/bootstrap/abi_assertion.go`

**Closest analogs:**
- `internal/ffi/types.go` - compile-time constants that define the Go-side ABI.
- `src/lib.rs` - Rust-side ABI constant that must remain numerically identical.

**Pattern to follow:**
- Keep the canary in `internal/bootstrap` so ordinary `go test ./internal/bootstrap` and downstream builds compile it.
- Import `github.com/amikos-tech/pure-simdjson/internal/ffi`; `internal/ffi` does not import `internal/bootstrap`, so this does not introduce an import cycle.
- Use constant array-length assertions, not runtime `init` panics, so drift fails at compile time.
- For `Version == "0.1.2"`, define a constant such as `abiVersionForBootstrapVersion_0_1_2 uint32 = 0x00010001` and assert both `ffi.ABIVersion - expected` and `expected - ffi.ABIVersion` are zero.

## Bootstrap Checksums Source

**Modify:** `internal/bootstrap/checksums.go`

**Closest analog:** Current file and `docs/releases.md`.

**Pattern to follow:**
- Keep `Checksums` empty in source.
- Comments should explain runtime `SHA256SUMS` resolution.
- Example comments may use `v0.1.2` but must not add real digest placeholders.

## Public Bootstrap Validation

**Existing file:** `.github/workflows/public-bootstrap-validation.yml`

**Closest analog:** Current workflow.

**Pattern to follow:**
- Dispatch after `release.yml` succeeds.
- Input is semver without leading `v`, for Phase 09.1: `0.1.2`.
- Treat all full R2 matrix jobs and GitHub fallback jobs as blocking validation evidence.

## Documentation Release Hedge

**Modify after validation:** `README.md`, `docs/benchmarks/results-v0.1.2.md`, `docs/bootstrap.md`, `CHANGELOG.md`

**Closest analogs:**
- Existing benchmark snapshot wording in `README.md`.
- Existing release boundary wording in `docs/benchmarks/results-v0.1.2.md`.
- Keep-a-Changelog section shape in `CHANGELOG.md`.

**Pattern to follow:**
- Before validation, `v0.1.2` remains upcoming-release evidence.
- After validation, remove the hedge and state `v0.1.2` as the current validated release/default-install path.
- Do not claim validation without workflow evidence.
