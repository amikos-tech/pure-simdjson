## L1 — cross-process locking coverage decision (locked)

The reviewer suggested a subprocess-based test (`exec.Command` running `go run`) to
exercise inter-process flock/LockFileEx semantics. Decision for v0.1:

**Document in this plan that multi-process lock behavior is covered by the
flock/LockFileEx semantics of the OS itself**, matching the pure-onnx precedent.
We will NOT add a subprocess-spawning test in this phase. Rationale:
- flock/LockFileEx correctness is an OS guarantee, not application code
- Subprocess tests are slow and flaky on CI (especially Windows path quoting)
- pure-onnx ships without one and has had no inter-process lock bugs in practice
- The intra-process concurrency test (TestConcurrentBootstrap, 8 goroutines)
  already exercises the lock-acquire loop, the retry-with-poll behavior, and
  the recheck-cached-artifact pattern — all of which are the application-level
  logic that could break

Add a comment block in bootstrap_test.go near TestConcurrentBootstrap documenting
this scope decision so future contributors don't rediscover the rationale.

```markdown
# Bootstrap and Distribution

## How It Works

`pure-simdjson` ships pre-built shared libraries for five platforms. On the first
call to `NewParser()`, the library automatically downloads, verifies (SHA-256),
caches, and loads the correct binary for your platform.

Download flow:
1. Check `PURE_SIMDJSON_LIB_PATH` — if set, load from that path (no download).
2. Check OS user cache (`~/Library/Caches/pure-simdjson/` on macOS, `~/.cache/pure-simdjson/` on Linux, `%LocalAppData%\pure-simdjson` on Windows). `PURE_SIMDJSON_CACHE_DIR` overrides the base.
3. If not cached: download from CloudFlare R2, verify SHA-256, cache with `0700` permissions (unix).
4. If R2 fails: fall back to GitHub Releases (unless `PURE_SIMDJSON_DISABLE_GH_FALLBACK=1`).
5. Load the verified artifact; verify ABI version post-load.

## Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `PURE_SIMDJSON_LIB_PATH` | Full path to a pre-built library. If set, bypasses all download and cache logic. Use for air-gapped environments. | — |
| `PURE_SIMDJSON_BINARY_MIRROR` | Override the R2 base URL. Must be HTTPS for non-loopback hosts. GitHub Releases fallback still fires on mirror failure. | `https://releases.amikos.tech/pure-simdjson` |
| `PURE_SIMDJSON_DISABLE_GH_FALLBACK` | Set to `1` to disable GitHub Releases fallback. Use in hermetic deployments where GitHub egress is blocked. | — |
| `PURE_SIMDJSON_CACHE_DIR` | Override the base directory used to cache downloaded artifacts. Useful for CI/CD runners with ephemeral or read-only HOME. | `$XDG_CACHE_HOME/pure-simdjson` (linux) / `~/Library/Caches/pure-simdjson` (darwin) / `%LocalAppData%\pure-simdjson` (windows) |

## Air-Gapped Deployment

Pre-fetch artifacts on a connected machine:

` ` ` bash
# Download all 5 platform artifacts to a local directory
pure-simdjson-bootstrap fetch --all-platforms --dest ./vendor-libs

# Transfer ./vendor-libs to the air-gapped machine, then:
export PURE_SIMDJSON_LIB_PATH=/path/to/vendor-libs/v0.1.0/linux-amd64/libpure_simdjson.so
` ` `

With `PURE_SIMDJSON_LIB_PATH` set, no network calls are made — ever.

To verify an offline bundle end-to-end:

` ` ` bash
pure-simdjson-bootstrap verify --all-platforms --dest ./vendor-libs
` ` `

## Corporate Firewall / Custom Mirror

If your environment blocks `releases.amikos.tech` but allows internal artifact mirrors:

` ` ` bash
# Set the mirror base URL. The library will use:
#   <MIRROR>/v<version>/<os>-<arch>/<libname>
export PURE_SIMDJSON_BINARY_MIRROR=https://artifacts.corp.internal/pure-simdjson
` ` `

If the mirror is also unreachable but `objects.githubusercontent.com` is allowed,
GitHub Releases fallback fires automatically (unless
`PURE_SIMDJSON_DISABLE_GH_FALLBACK=1`).

## Offline Pre-Fetch with the CLI

The `pure-simdjson-bootstrap` CLI pre-downloads artifacts for CI caching, offline
bundle building, or corporate mirrors:

` ` ` bash
# Install the CLI
go install github.com/amikos-tech/pure-simdjson/cmd/pure-simdjson-bootstrap@latest

# Download for all platforms to a custom directory (shows per-platform progress)
pure-simdjson-bootstrap fetch --all-platforms --dest ./libs

# Download for a specific platform
pure-simdjson-bootstrap fetch --target linux/amd64 --dest ./libs

# Verify cached artifacts (current platform; OS user cache)
pure-simdjson-bootstrap verify

# Verify an offline bundle end-to-end

## GitHub Release Asset Naming

Release artifacts are published to both CloudFlare R2 and GitHub Releases. R2 uses a
directory layout per platform, so all files can share the same base filename. GitHub
Release assets live in a flat namespace, so the filenames are platform-tagged to
avoid collision:

| Source | linux/amd64 | linux/arm64 | darwin/amd64 | darwin/arm64 | windows/amd64 |
|--------|-------------|-------------|--------------|--------------|----------------|
| R2 (dir) | `libpure_simdjson.so` | `libpure_simdjson.so` | `libpure_simdjson.dylib` | `libpure_simdjson.dylib` | `pure_simdjson-msvc.dll` |
| GitHub (flat) | `libpure_simdjson-linux-amd64.so` | `libpure_simdjson-linux-arm64.so` | `libpure_simdjson-darwin-amd64.dylib` | `libpure_simdjson-darwin-arm64.dylib` | `pure_simdjson-windows-amd64-msvc.dll` |

The bootstrap library handles this transparently; you only see the R2-style name
on disk after the library caches the download.

## Verifying Artifact Integrity (Cosign)

Release artifacts are signed with cosign keyless OIDC signing. Verification is
optional but recommended for sensitive deployments.

` ` ` bash
TAG=v0.1.0
OS=linux
ARCH=amd64
BASE_URL="https://releases.amikos.tech/pure-simdjson/${TAG}"
LIB="libpure_simdjson.so"

curl -LO "${BASE_URL}/${OS}-${ARCH}/${LIB}"
curl -LO "${BASE_URL}/${OS}-${ARCH}/${LIB}.sig"
curl -LO "${BASE_URL}/${OS}-${ARCH}/${LIB}.pem"

cosign verify-blob \
  --signature "${LIB}.sig" \
  --certificate "${LIB}.pem" \
  --certificate-identity "https://github.com/amikos-tech/pure-simdjson/.github/workflows/release.yml@refs/tags/${TAG}" \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com" \
  "${LIB}"
` ` `

SHA-256 verification is always performed by the library before loading (no
configuration required). Cosign adds a provenance layer for users who want to
verify the release pipeline was not compromised.

## Retry and Error Behavior

The bootstrap pipeline retries transient failures (HTTP 429, 5xx, GitHub 403
rate-limit) with Full-Jitter exponential backoff (capped at 8 seconds). Context
cancellation propagates within milliseconds. On blocked networks, bootstrap
failures are memoized per-process for 30 seconds so repeated `NewParser()` calls
fail fast.

If all download sources fail, the error message includes a hint:

` ` `
bootstrap failed (set PURE_SIMDJSON_LIB_PATH to bypass): all sources failed: ...
` ` `

Set `PURE_SIMDJSON_LIB_PATH` to a manually downloaded artifact to bypass the
download pipeline entirely.
```
