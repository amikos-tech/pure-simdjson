# Bootstrap and Distribution

## How It Works

`pure-simdjson` ships pre-built shared libraries for five platforms. On the first call to `NewParser()`, the library automatically downloads, verifies (SHA-256), caches, and loads the correct binary for your platform.

Download flow:
1. Check `PURE_SIMDJSON_LIB_PATH` — if set, load from that path (no download).
2. Check OS user cache (`~/Library/Caches/pure-simdjson/` on macOS, `~/.cache/pure-simdjson/` on Linux, `%LocalAppData%\pure-simdjson` on Windows).
3. If not cached: download from CloudFlare R2, verify SHA-256, cache with `0700` permissions (unix).
4. If R2 fails: fall back to GitHub Releases.
5. Load the verified artifact; verify ABI version post-load.

## Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `PURE_SIMDJSON_LIB_PATH` | Full path to a pre-built library. If set, bypasses all download and cache logic. Use for air-gapped environments. | — |
| `PURE_SIMDJSON_BINARY_MIRROR` | Override the R2 base URL. Must be HTTPS for non-loopback hosts. GitHub Releases fallback still fires on mirror failure. | `https://releases.amikos.tech/pure-simdjson` |
| `PURE_SIMDJSON_DISABLE_GH_FALLBACK` | Set to `1` to disable GitHub Releases fallback. Use in hermetic deployments where GitHub egress is blocked. | — |

## Air-Gapped Deployment

Pre-fetch artifacts on a connected machine:

```bash
# Download all 5 platform artifacts to a local directory
pure-simdjson-bootstrap fetch --all-platforms --dest ./vendor-libs

# Transfer ./vendor-libs to the air-gapped machine, then:
export PURE_SIMDJSON_LIB_PATH=/path/to/vendor-libs/v0.1.0/linux-amd64/libpure_simdjson.so
```

With `PURE_SIMDJSON_LIB_PATH` set, no network calls are made — ever.

## Corporate Firewall / Custom Mirror

If your environment blocks `releases.amikos.tech` but allows internal artifact mirrors:

```bash
# Set the mirror base URL. The library will use:
#   <MIRROR>/v<version>/<os>-<arch>/<libname>
export PURE_SIMDJSON_BINARY_MIRROR=https://artifacts.corp.internal/pure-simdjson
```

If the mirror is also unreachable but `objects.githubusercontent.com` is allowed, GitHub Releases fallback fires automatically (unless `PURE_SIMDJSON_DISABLE_GH_FALLBACK=1`).

## Offline Pre-Fetch with the CLI

The `pure-simdjson-bootstrap` CLI pre-downloads artifacts for CI caching, offline bundle building, or corporate mirrors:

```bash
# Install the CLI
go install github.com/amikos-tech/pure-simdjson/cmd/pure-simdjson-bootstrap@latest

# Download for all platforms to a custom directory
pure-simdjson-bootstrap fetch --all-platforms --dest ./libs

# Download for a specific platform
pure-simdjson-bootstrap fetch --target linux/amd64 --dest ./libs

# Verify cached artifacts
pure-simdjson-bootstrap verify

# List platforms and cache status
pure-simdjson-bootstrap platforms

# Show version information
pure-simdjson-bootstrap version
```

## Verifying Artifact Integrity (Cosign)

Release artifacts are signed with cosign keyless OIDC signing. Verification is optional but recommended for sensitive deployments.

```bash
TAG=v0.1.0
OS=linux
ARCH=amd64
BASE_URL="https://releases.amikos.tech/pure-simdjson/${TAG}"
LIB="libpure_simdjson.so"

curl -LO "${BASE_URL}/${OS}-${ARCH}/${LIB}"
curl -LO "${BASE_URL}/${OS}-${ARCH}/${LIB}.sig"
curl -LO "${BASE_URL}/${OS}-${ARCH}/${LIB}.pem"

cosign verify-blob \
  --signature "${LIB}.sig" \
  --certificate "${LIB}.pem" \
  --certificate-identity "https://github.com/amikos-tech/pure-simdjson/.github/workflows/release.yml@refs/tags/${TAG}" \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com" \
  "${LIB}"
```

SHA-256 verification is always performed by the library before loading (no configuration required). Cosign adds a provenance layer for users who want to verify the release pipeline was not compromised.

## Retry and Error Behavior

The bootstrap pipeline retries transient failures (HTTP 429, 5xx) with exponential backoff (capped at 8 seconds). Context cancellation propagates within milliseconds.

If all download sources fail, the error message includes a hint:

```
bootstrap failed (set PURE_SIMDJSON_LIB_PATH to bypass): all sources failed: ...
```

Set `PURE_SIMDJSON_LIB_PATH` to a manually downloaded artifact to bypass the download pipeline entirely.
