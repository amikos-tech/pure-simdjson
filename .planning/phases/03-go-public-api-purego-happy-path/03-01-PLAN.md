<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo-local file system -> purego loader | Wrong path resolution or bare-name loading would produce nondeterministic library selection and Windows DLL hijack risk |
| Go-owned buffers -> native string-copy helpers | Buffer-backed purego calls can observe reclaimed Go memory if liveness is not preserved explicitly |
| native ABI results -> Go errors | Wrong constant mapping or stale diagnostic reuse would misreport failures to callers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | T | local loader | mitigate | Resolve only full explicit paths, preserve attempted-path order, and forbid recursive scans or bootstrap behavior in Phase 3 |
| T-03-01-02 | T | purego string-copy helpers | mitigate | Add `runtime.KeepAlive(...)` immediately after native copy calls that depend on Go-owned buffers or receiver-owned handles |
| T-03-01-03 | T | Go FFI mirrors | mitigate | Mirror `include/pure_simdjson.h` exactly for handles, value views, error codes, and implemented value kinds before higher-level code is written |
| T-03-01-04 | I | structured error wrapping | mitigate | Build errors from the failing call's immediate native diagnostics only and treat empty text/sentinel offsets as absent detail |
</threat_model>
