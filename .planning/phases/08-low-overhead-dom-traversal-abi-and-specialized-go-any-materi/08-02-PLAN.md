<interfaces>
Internal frame layout to implement exactly in Rust/C++:
```c
typedef struct psdj_internal_frame_t {
  uint32_t kind;
  uint32_t flags;
  uint32_t child_count;
  uint32_t reserved;
  const uint8_t *key_ptr;
  size_t key_len;
  const uint8_t *string_ptr;
  size_t string_len;
  int64_t int64_value;
  uint64_t uint64_value;
  double float64_value;
} psdj_internal_frame_t;
```

Internal export signature:
```c
pure_simdjson_error_code_t psdj_internal_materialize_build(
    const pure_simdjson_value_view_t *view,
    const psdj_internal_frame_t **out_frames,
    size_t *out_frame_count
);
```

Frame semantics:
- `kind` uses existing `PURE_SIMDJSON_VALUE_KIND_*` numeric values.
- Frames are preorder. Container frames set exact `child_count`.
- Object member frames carry `key_ptr/key_len`; array member frames carry `key_ptr == NULL` and `key_len == 0`.
- String frames carry `string_ptr/string_len`; Go must copy before returning.
- Integral and float frames set only the matching numeric field.
- `reserved` must be zero on every frame.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Go/internal FFI -> Rust registry | Untrusted view structs cross into Rust and must be generation/tag/reserved-bit checked. |
| Rust registry -> C++ DOM | Validated doc pointer and json index cross into C++ traversal. |
| C++ borrowed bytes -> Go materializer | Frame key/string pointers remain borrowed and doc-lifetime-bound. |
| Internal export -> public header | Dynamic symbols used by Go must not become public C ABI promises. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering/Denial of Service | `psdj_internal_frame_t` borrowed pointers | mitigate | Store frames in doc-owned native scratch, validate views once through `with_resolved_view`, and leave copying to Plan 03 before values escape. |
| T-08-02 | Information Disclosure/Tampering | `include/pure_simdjson.h` | mitigate | Exclude `psdj_internal_*` from cbindgen and verify with `make verify-contract`. |
| T-08-03 | Tampering | numeric frame fields | mitigate | Emit exact `kind` tags plus split numeric fields; test `uint64` max and typed error propagation. |
| T-08-04 | Denial of Service | native frame scratch | accept | This plan creates frame scratch only; Plan 04 checks benchmark reuse does not cache final Go trees. |
| T-08-05 | Denial of Service | `psdj_internal_materialize_build` signature | mitigate | Use status return plus pointer out-params only, no callbacks, no struct-by-value return, and contract verification. |
</threat_model>
