<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internal frame slice -> Go values | Borrowed frame/key/string memory is read inside one call and copied before return. |
| public Element -> internal binding | Public wrapper state controls whether an internal FFI call is allowed. |
| internal symbol lookup -> Go package | Dynamic symbol is internal and unexported from public Go API. |
| concurrent Close/materialize -> borrowed native memory | `Doc` mutex guards frame-span consumption from concurrent close or rebuild. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering/Denial of Service | `materializer_fastpath.go` | mitigate | Copy keys/strings into Go values, hold `doc.mu` while reading borrowed frames, and defer `runtime.KeepAlive(doc)` after a successful frame fetch. |
| T-08-02 | Information Disclosure/Tampering | Go public API | mitigate | Keep only unexported `fastMaterializeElement`; grep for absence of `Interface` methods. |
| T-08-03 | Tampering | numeric and oversized-literal materialization boundary | mitigate | Use frame `Kind` plus split numeric fields for materialized values and run `TestFastMaterializerNumericSemantics` plus `TestFastMaterializerOversizedLiteralParseRejected` to preserve parse-time rejection for oversized literals. |
| T-08-04 | Denial of Service | materializer benchmark path | mitigate | Build fresh maps/slices/strings on every call; do not cache final Go trees. |
| T-08-05 | Denial of Service | Go FFI frame layout | mitigate | Add `unsafe.Sizeof` and `unsafe.Offsetof` layout tests for `InternalFrame`; fail `ErrInternal` on stream desync. |
</threat_model>
