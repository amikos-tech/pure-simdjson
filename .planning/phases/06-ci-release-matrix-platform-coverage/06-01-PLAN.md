Implement `.github/actions/setup-rust/action.yml` so it installs the repo-pinned Rust toolchain from `rust-toolchain.toml`, optional target triples, and any linux-only helper tooling required by downstream actions.

Implement `.github/actions/build-shared-library/action.yml` with inputs at minimum:
- `target`
- `goos`
- `goarch`
- `runner_kind` (`linux`, `darwin`, `windows`)
- `manylinux_image` (optional)
- `use_zigbuild` (default false)
- `source_date_epoch`
- `verify_reproducible` (default false)

The action must:
- install Rust from `rust-toolchain.toml`, not `@stable`
- export `CARGO_INCREMENTAL=0`
- build with `cargo build --release --locked --target <triple>`
- set deterministic flags before every build:
  - common: `RUSTFLAGS="--remap-path-prefix=${GITHUB_WORKSPACE}=. -C codegen-units=1 -C debuginfo=0 -C strip=symbols"`
  - linux append: `-C link-arg=-Wl,--build-id=none`
  - darwin append: `-C link-arg=-Wl,-no_uuid`
  - windows append: `-C link-arg=/Brepro`
- require `SOURCE_DATE_EPOCH=<git log -1 --pretty=%ct>` from the caller and forward it into cargo
- build `libpure_simdjson.so` for linux targets
- build `libpure_simdjson.dylib` for darwin targets
- build local `pure_simdjson.dll` for windows targets
- output the absolute built library path
- when `verify_reproducible=true`, clean and build the same target twice into separate temp directories, hash both outputs, print both digests, and fail if they differ before packaging continues

Implement `scripts/release/package_shared_artifact.sh` so it accepts:
- `--input <built-library-path>`
- `--goos <goos>`
- `--goarch <goarch>`
- `--rust-target <triple>`
- `--version <semver-without-v>`
- `--out-dir <dir>`

The script must emit:
- an R2/cache-path file named from `internal/bootstrap/url.go::PlatformLibraryName`
- a GitHub asset copy named from the bootstrap GitHub naming contract:
  - `libpure_simdjson-linux-amd64.so`
  - `libpure_simdjson-linux-arm64.so`
  - `libpure_simdjson-darwin-amd64.dylib`
  - `libpure_simdjson-darwin-arm64.dylib`
  - `pure_simdjson-windows-amd64-msvc.dll`
- one manifest entry suitable for aggregation into a single release manifest with exact shape:
  - top-level: `{ "version": "<semver-without-v>", "entries": [ ... ] }`
  - each `entries[]` object must record `goos`, `goarch`, `rust_target`, `r2_key`, `github_asset_name`, `local_path`, and `sha256`
- compute `entries[].sha256` from the exact packaged R2/cache-path bytes written by this script, not from a different staging path or a later recomputation step
- record `entries[].local_path` as the packaged R2/cache-path file path that downstream staging and verification steps will consume

Wrap that script in `.github/actions/package-shared-artifact/action.yml`.

Implement `scripts/release/update_bootstrap_release_state.py` so it rewrites only `internal/bootstrap/version.go` and `internal/bootstrap/checksums.go` from a manifest file generated by the packaging step.

The script must:
- accept `--manifest <path>` and `--version <semver-without-v>`
- require the manifest to have exact shape `{ "version": "<semver-without-v>", "entries": [ ... ] }`, where every `entries[]` item contains `goos`, `goarch`, `rust_target`, `r2_key`, `github_asset_name`, `local_path`, and `sha256`
- fail if the top-level manifest version, any `entries[].r2_key`, or any platform tuple disagrees with `--version`
- sort entries in this exact order:
  1. `linux/amd64`
  2. `linux/arm64`
  3. `darwin/amd64`
  4. `darwin/arm64`
  5. `windows/amd64`
- write `const Version = "<version>"` in `internal/bootstrap/version.go`
- fail if any entry is missing `sha256`, if any `sha256` is not 64 lowercase hex characters, or if any checksum key would be duplicated
- write checksum keys exactly in the bootstrap format:
  - `v<version>/linux-amd64/libpure_simdjson.so`
  - `v<version>/linux-arm64/libpure_simdjson.so`
  - `v<version>/darwin-amd64/libpure_simdjson.dylib`
  - `v<version>/darwin-arm64/libpure_simdjson.dylib`
  - `v<version>/windows-amd64/pure_simdjson-msvc.dll`
- write the checksum values directly from `entries[].sha256`; do not emit placeholders and do not silently re-hash `local_path`
- preserve the surrounding comment blocks in both Go files

Add `scripts/release/test_update_bootstrap_release_state.py` that copies `internal/bootstrap/version.go` and `internal/bootstrap/checksums.go` into a temporary workspace, creates a temporary manifest with all five entries, runs the rewrite script against that temp workspace, and asserts:
- `internal/bootstrap/version.go` contains the requested version
- `internal/bootstrap/checksums.go` contains all five expected keys
- `internal/bootstrap/checksums.go` contains the exact known SHA-256 values from the temporary manifest for those five keys
- rerunning the script on the same manifest is idempotent
- the script fails when an entry is missing `sha256` or provides a malformed digest
- the real repository working tree is unchanged after the test completes
