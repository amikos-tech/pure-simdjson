Create `scripts/release/assemble_staged_release_tree.sh` that accepts `--version <semver-without-v>`, `--manifest-dir <dir>`, `--artifact-dir <dir>`, and `--staged-root <dir>`. It must:
- require exactly five manifest rows and exactly these cache-layout outputs:
  - `v<version>/linux-amd64/libpure_simdjson.so`
  - `v<version>/linux-arm64/libpure_simdjson.so`
  - `v<version>/darwin-amd64/libpure_simdjson.dylib`
  - `v<version>/darwin-arm64/libpure_simdjson.dylib`
  - `v<version>/windows-amd64/pure_simdjson-msvc.dll`
- fail fast if any artifact or manifest row is missing
- copy from the packaged workflow artifacts into that exact directory tree so the same staged root can feed the loopback bootstrap smoke and later the R2 publish step

Create a dedicated Go smoke program at `tests/smoke/go_bootstrap_smoke.go` that:
- imports `github.com/amikos-tech/pure-simdjson`
- calls `purejson.NewParser()`
- parses the literal JSON `42`
- verifies the root value is `42`
- exits non-zero on any error

Create `tests/smoke/ffi_export_surface.c` as the explicit CI-04 ABI-load harness. It must:
- load the staged shared library through the platform loader (`dlopen`/`dlsym` on unix, `LoadLibraryW`/`GetProcAddress` on Windows)
- resolve every public function declared in `include/pure_simdjson.h` for the Phase 5 surface, not just list exports
- invoke each resolved symbol once with safe inputs and assert either the happy-path result or the expected typed error for deliberately invalid calls
- cover at minimum the parser lifecycle, document root, typed accessors, array/object iterator helpers, ABI/version diagnostics, and any exported cleanup helpers required by the current header surface
- fail if any header-declared symbol cannot be resolved or is resolved but never called

Create `scripts/release/run_native_smoke.sh` that accepts a staged artifact path and platform tuple, then performs the native release gate in this exact order:
1. run platform-native supplemental audits (`nm`/`objdump`, `codesign --verify` + `file`, or `dumpbin /EXPORTS`)
2. compile and run `tests/smoke/ffi_export_surface.c` against the staged artifact so every exported ABI symbol is loaded and invoked once
3. compile and run `tests/smoke/minimal_parse.c` against the same staged artifact so one parse round-trip is still proven after the ABI-load gate
CI-04 is not satisfied unless all three stages pass.

Create `scripts/release/run_go_packaged_smoke.sh` that:
- accepts `--staged-root <dir>` and `--version <semver-without-v>`
- serves the staged artifact tree over loopback HTTP
- explicitly `unset PURE_SIMDJSON_LIB_PATH`
- sets `PURE_SIMDJSON_BINARY_MIRROR=http://127.0.0.1:<port>/pure-simdjson`
- sets `PURE_SIMDJSON_DISABLE_GH_FALLBACK=1`
- sets `PURE_SIMDJSON_CACHE_DIR` to a temporary empty directory
- verifies the exact env-var contract before running by grepping `internal/bootstrap/bootstrap.go` for `PURE_SIMDJSON_BINARY_MIRROR` and `PURE_SIMDJSON_DISABLE_GH_FALLBACK`
- runs `go run ./tests/smoke/go_bootstrap_smoke.go`

Create `scripts/release/run_alpine_smoke.sh` that:
- accepts `--image-ref <alpine:latest@sha256:<resolved-digest>>`
- runs the entire Alpine validation inside that exact image reference and refuses to proceed if the workflow passes a different container ref
- installs Go, Rust, clang/g++, and build essentials
- runs `cargo build --release`
- sets `PURE_SIMDJSON_LIB_PATH=<repo>/target/release/libpure_simdjson.so`
- runs `go run ./tests/smoke/go_bootstrap_smoke.go`

Do not use `PURE_SIMDJSON_LIB_PATH` in the generic packaged-artifact smoke script; that path must exercise the real bootstrap mirror flow. `PURE_SIMDJSON_LIB_PATH` remains Alpine-only per D-01 and the Phase 5 contract, and the Alpine image reference must stay singular across script and workflow (`alpine:latest@sha256:<resolved-digest>`, not a floating tag in one place and a pinned digest in another).
