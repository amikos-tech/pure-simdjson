  - `SHA256SUMS`
- use `cosign sign-blob --yes --output-signature <file>.sig --output-certificate <file>.pem <file>` for each blob
- verify those signatures in the workflow before upload with `cosign verify-blob --certificate-identity "https://github.com/amikos-tech/pure-simdjson/.github/workflows/release.yml@refs/tags/${{ github.ref_name }}" --certificate-oidc-issuer "https://token.actions.githubusercontent.com"`

  - `SHA256SUMS`
  - artifact `.sig` / `.pem`
- refuse to upload if the R2 prefix already exists
- publish the platform-tagged GitHub Release assets for:
  - every raw shared-library asset
  - `SHA256SUMS`
  - every generated `.sig`
  - every generated `.pem`
- enable generated release notes so GitHub Releases mirrors the same signed, checksummed release set as R2

Implement `scripts/release/publish_r2.sh` so it accepts:
- `--version <tag-with-v>`
- `--staged-root <dir>`
- required R2 credentials through env vars

The script must check for an existing prefix before upload and fail fast if the prefix already contains objects.

| Boundary | Description |
|----------|-------------|
| prep commit -> publish tag | publish must only happen from a source tree whose bootstrap metadata already matches the artifact set |
| CI artifacts -> public R2/GitHub release | only signed, checksum-coherent artifacts may cross this boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05-01 | T | checksum/tag coherence | mitigate | tag workflow recomputes the same manifest contract and aborts on any entries[].sha256 drift from committed state |
| T-06-05-02 | T | public artifact provenance | mitigate | cosign-sign every artifact plus SHA256SUMS and verify inside CI before upload |
| T-06-05-03 | T | R2 release immutability | mitigate | fail upload if the destination prefix already exists |
</threat_model>
