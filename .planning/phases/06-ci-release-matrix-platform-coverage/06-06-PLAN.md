Create `docs/releases.md` as the authoritative Phase 6 runbook. It must document:
- the exact pre-tag `release-prepare.yml` flow:
  1. dispatch `release-prepare.yml` with `version=<semver-without-v>`
  2. inspect the uploaded manifest and staged release-tree artifacts
  3. open a PR from `release-prep/v<version>` into `main`
  4. merge that PR
  5. create annotated tag `v<version>` on the merged `main` commit only
- the exact tag-push `release.yml` flow, including the `git merge-base --is-ancestor "$GITHUB_SHA" "origin/main"` guard that rejects off-main tags
- required GitHub secrets / vars for R2 and release publication
- the exact published URL layout for raw artifacts and `SHA256SUMS`
- the cosign verification commands for one raw library artifact and for `SHA256SUMS`
- the keyless GitHub OIDC signing mode and the exact certificate identity / issuer used during verification
- the exact macOS Gatekeeper workaround for downloaded dylibs: `xattr -d com.apple.quarantine <path-to-dylib>` before retrying the load when the ad-hoc-signed artifact is blocked; note that this operator step is the one referenced by Phase 6 Success Criterion 5 and `06-VALIDATION.md`
- the Phase `06.1` follow-up boundary: fresh-machine public validation happens after publish, not during Phase 6

Create `scripts/release/check_readiness.sh` with a `--strict` mode. In strict mode it must fail if:
- `python3 scripts/release/assert_prepared_state.py --check-source --version <semver-without-v>` fails
- `git fetch origin main --depth=1 && git merge-base --is-ancestor HEAD origin/main` fails
- `.github/workflows/release-prepare.yml` or `.github/workflows/release.yml` is missing
- `docs/releases.md` is missing

Update `docs/bootstrap.md` so it links directly to `docs/releases.md` where Phase 6 publish-time behavior and signature verification now live, and include the same macOS operator guidance for published dylibs: `xattr -d com.apple.quarantine <path-to-dylib>` before retrying the load if Gatekeeper blocks the download.
