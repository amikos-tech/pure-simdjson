  # NOTE (Phase 10 WARN-4): hashFiles excludes the simdjson submodule SHA. A submodule pointer change
  # without a CMakeLists.txt edit silently serves a stale target/release/. Mitigation: when bumping

  uses: actions/cache@0400d5f644dc74513175e3cd8d07132dd4860809   # v4.2.4
  with:

    key: pr-bench-rust-${{ runner.os }}-${{ hashFiles('Cargo.lock', 'rust-toolchain.toml', 'third_party/simdjson/CMakeLists.txt') }}
    restore-keys: |
      pr-bench-rust-${{ runner.os }}-

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `pull_request` event source | The PR's HEAD SHA. Fork PRs run with limited (read-only) `GITHUB_TOKEN`; same-repo PRs run with full `pull-requests: write`. The workflow MUST tolerate both. |
| third-party action → runner | Each pinned action runs in the runner with the `GITHUB_TOKEN` and the workspace contents. SHA pinning prevents tag-mutation supply-chain swaps. |
| `actions/cache` → cross-workflow data flow | The push-on-main workflow writes a cache key visible to PR workflows on the same default branch. PR workflows read but never write. |
| sticky-comment action → PR comments API | Posts/edits PR comments. Needs `pull-requests: write` token. Fork PRs lack this; `continue-on-error: true` swallows the 403. |
