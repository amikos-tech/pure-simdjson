pr-bench-summary/

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `--baseline <path>` arg → orchestrator | Path comes from the workflow YAML (Plan 03), which gets it from `actions/cache/restore` — the cache content was written by the push-on-main workflow on `refs/heads/main`. Trust is rooted in GitHub's branch-scoped cache isolation. |
| `go test -bench` stdout → head.bench.txt | Output is the PR's own runner producing benchmark numbers. The PR head can affect the values but cannot affect the file format (Go's testing harness owns it). |
| stage_dir → out_dir atomic mv | Stage dir is `mktemp -d` under the same parent as out_dir (same FS), so `mv` is atomic. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-06 | T (Tampering) | A malicious PR injects benchmark code that prints fake `+0.00%` rows and confuses benchstat. | accept | benchstat is invoked on RAW `.bench.txt` from `go test -bench` — it computes its own deltas. The benchstat program is the trust anchor. A PR can't fake benchstat's own output without compromising `golang.org/x/perf` itself, which is beyond Phase 10's scope. |
| T-10-07 | T (Tampering) | The orchestrator writes the canonical baseline cache from a PR, poisoning future PR comparisons. | mitigate | Acceptance criteria: `! grep -q "actions/cache" scripts/bench/run_pr_benchmark.sh`. The cache write step lives ONLY in Plan 03's main-baseline workflow. Per RESEARCH § Threat Register / "Cache poisoning". |
| T-10-08 | I (Information Disclosure) | Bash `set -x` or echoed env exposes secrets in workflow logs. | mitigate | Orchestrator uses `set -euo pipefail` (no `-x`). It does not echo `env`, does not call `printenv`, does not source `.env` files. Every echoed string is a known constant or a path. |
| T-10-09 | D (DoS) | A PR puts `go test -bench` inside an infinite loop or extremely large fixture, blowing the 10-minute budget. | mitigate | `-timeout 600s` flag in the orchestrator caps `go test` execution at exactly 10 minutes. The workflow YAML (Plan 03) adds a 12-minute `timeout-minutes` job-level cap as defense in depth. |
| T-10-10 | E (Elevation) | A PR puts shell-injection content into a fixture filename or env variable that the orchestrator interprets. | mitigate | The orchestrator's only variable-interpolated paths come from named CLI flags (`--baseline`, `--out-dir`); both are double-quoted at every use. No `eval`, no unquoted globs, no `IFS` mutation. |
</threat_model>
