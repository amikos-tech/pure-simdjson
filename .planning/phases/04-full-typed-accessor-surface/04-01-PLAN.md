<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public value views -> Rust runtime validation | Malformed or stale transport structs must fail as `ERR_INVALID_HANDLE`, not become dangling native pointers |
| borrowed native string bytes -> Rust-owned ABI buffers | String copy-out must never depend on C++ allocator symmetry across DLL boundaries |
| native string allocations -> Go consumers | Successful string reads now include explicit native memory release requirements |
| C++ bridge errors -> public ABI status codes | Wrong numeric/string error mapping would break the locked Go error semantics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | T | descendant-view validation | mitigate | Lock one tag namespace (`PSDJROOT`, `PSDJDESC`), reject unknown tags/non-zero reserved state, and force descendant access through `doc + json_index` helpers instead of trusting raw caller pointers |
| T-04-01-02 | T | string allocation/free | mitigate | Keep string copy-out allocator ownership in Rust, accept only buffers created by that path in `bytes_free`, and prove empty/non-empty round-trips in ABI tests |
| T-04-01-03 | I | error-code mapping | mitigate | Keep bridge-side mappings aligned with the Phase 1 contract for wrong-type, overflow, precision-loss, and invalid JSON cases |
</threat_model>
