<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public value views -> Rust runtime validation | Malformed or stale transport structs must fail as `ERR_INVALID_HANDLE`, not become dangling native pointers |
| native string allocations -> Go consumers | String-copy success paths now include explicit native memory release requirements |
| C++ bridge errors -> public ABI status codes | Wrong numeric/string error mapping would break the locked Go error semantics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | T | value-view validation | mitigate | Generalize validation beyond root-only, reject unknown tags/non-zero reserved state, and confirm the owning doc is still live before using any native element pointer |
| T-04-01-02 | T | string allocation/free | mitigate | Pair `element_get_string` with `bytes_free`, copy bytes before returning to Go, and add direct integration coverage for empty/non-empty strings |
| T-04-01-03 | I | error-code mapping | mitigate | Keep bridge-side `map_error(...)` aligned with the Phase 1 contract for wrong-type, overflow, precision-loss, and invalid JSON cases |
</threat_model>
